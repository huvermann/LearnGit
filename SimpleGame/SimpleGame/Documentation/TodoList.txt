  - 6.0: Beampoint
	  + 6.1: Tor mit xy-Koordinaten auf Map, damit man komfortabel von Map zu Map reisen kann.
         mit BeamPoint(x,y) soll man die Spielerposition innerhalb der Map konfigurieren.
		Bemerkung:

		Beampoints werden auf der Map im Objects-Layer angelegt.
		Jeder Beampoint muss einen eindeutigen Namen haben.
		Die xy-Position des Objekt-Rechtecks wird als Zielposition für den Spieler definiert
		über die globale Beampoint-Registry kann man alle Beampoints abrufen.
		Die beampoint.beam() methode erzeugt ein Change-View-Event. Die Positionsänderung wird dann vom Eventhändler erledigt.

		Jeder Beampoint im Spiel kann über die Beampoint-Registry angesprungen werden:
		
		Beispiel:
		beamService = ServiceLocator.getGlobalServiceInstance(ServiceNames.BeamPoints)
        beamService.beam("Ziel3")

  - 7.0: Impl.: Sprite Basisklassen
  7.1: + Sprite läuft hin- und her

  Neue Sprite-Basisklasse, die folgende properties aus der TMX-Map-Datei:
- Score: Anzahl der Punkte bei Berührung
- Style: Style-Klasse, die die Animationskonfiguration enthält
- KillSprite: True oder 1, wenn das Sprite bei Berührung verschwinden soll
- KillPlayer: True, wenn der Spieler bei Berührung sterben soll
- Behaviour: Bei Berührung wird ein bestimmtes programmiertes Verhalten ausgelöst.
- Item: Bei Berührung wird der Inventory des Spielers ein Item hinzugefügt.
- Energie: Es wird dem Spieler Energie hinzugefügt.
- Sound: Bei Berührung wird eine Sound-Datei abgespielt.
- Itelligence: Programmierte Bewegungen und Verhalten währen des Spiels.

- 8.0: Das Gageway-Sprite als Basisklasse implementieren.
     + 8.1 Beim Gateway soll man einfach nur die Grafik austausen damit Gateways mit unterschiedlichem Aussehen erstellt werden können.

Bemerkung zu 8.0 und 8.1.
Ein Gateway-Sprite muß einfach nur das TeleporterBehavior als Behavior-Eigenschaft verwenden. Die Grafik wird ganz normal über die Style-Eigenschaft festgelegt.
Ein Sprite mit dem TeleporterBehavior verwendet die Properties ViewPoint oder Beam-Point.
Beampoints (Ziele für den Teleporter) werden auf der Objekt-Layer-Ebene als Typ BeamPoint angelegt.
Siehe auch Manual_GameDesigner.docx
